## What It Is

A single Rust binary (1.8 MB static musl) that handles the full antenna-to-browser pipeline for both aviation (ADS-B/Mode-S) and maritime (AIS) tracking. Replaces readsb + dump1090 + AIS-catcher + hpr-atlas.

## Current State

### ✅ Production (Aggregator Mode)
- Running on port 30004, 2500+ feeders connected
- 99.96% position accuracy
- AIS: 34,377 vessels from atlas NMEA feed
- Self-learning receiver map (2335 receivers)
- 109k cached routes
- tar1090 frontend served at :8088

### ✅ Working (Edge Mode — today's achievement)
- Full stack on Orange Pi 2 (ARM, 2GB RAM)
- RTL-SDR → built-in demodulator → decode → tar1090 map
- 52 aircraft tracked, 7 with positions
- Docker container deployment
- 17.5% CPU, 15 MB RAM

### ⚠️ Needs Work
| Area | Status | Issue |
|------|--------|-------|
| 1090 MHz demod thresholds | Works but noisy | Accepts too many weak frames → inflated aircraft count (349 vs 52 real). Fixed by raising conf threshold. Still shows fewer positions than readsb (~7 vs ~40). |
| AIS 162 MHz demod | Architecture correct, 0 decoded | CRC fails on all 54 detected packets. ISI from ÷3 average decimation. Need proper Blackman-Harris FIR. |
| pos_reliable | Lowered to 1 for edge | At 2, no positions show (too few messages from weak SDR). Need adaptive threshold based on message rate. |
| Edge → Aggregator feed | Not wired | `--feed-to` upstream client not yet implemented. |

## Architecture

```
┌────────────────────────────────────────────────┐
│  hpr-skygate (1.8 MB static binary)           │
├────────────────────────────────────────────────┤
│  SDR Layer (--sdr-device)                      │
│    rtl_sdr subprocess → Demod1090              │
│    Preamble → Manchester → CRC → Beast binary  │
├────────────────────────────────────────────────┤
│  Ingest (mio epoll, single-thread)            │
│    Beast TCP :30004 ← 2500 feeders            │
│    OR local SDR self-connect                   │
├────────────────────────────────────────────────┤
│  Decode                                        │
│    Mode-S DF0-21 parse                         │
│    ADS-B TC1-31 (ident, position, velocity)    │
│    CPR global + relative, pos_reliable         │
│    Speed check, track direction validation     │
│    Per-receiver slot pairing                   │
├────────────────────────────────────────────────┤
│  AIS (--ais-sources)                           │
│    TCP NMEA reader with auto-reconnect         │
│    normalizeNMEA, dedup, FragmentAssembler     │
│    VesselStore (30min TTL)                     │
├────────────────────────────────────────────────┤
│  Output                                        │
│    HTTP :8088/8089 (tar1090 + JSON API)        │

│    WebSocket binary push (planned)             │
└────────────────────────────────────────────────┘
```

## What Was Done Today

1. **AIS reader hardened** — normalize NMEA, 64KB buffer, per-source stats
2. **Full AIS feed tested** — 34,377 vessels from atlas NMEA fanout (inst4)
3. **Phase C: SDR demod absorbed** — 1090 MHz + 162 MHz modules in skygate
4. **OPi2 edge testing** — cross-compiled ARM, real SDR, real aircraft decoded
5. **AIS demod rewrite** — ais-catcher architecture (channel split, CIC5, 5-decoder brute-force timing, state machine HDLC). Still 0 CRC OK.
6. **1090 edge full stack** — antenna → browser working in Docker on OPi2

## Lessons Learned (Never Again)

1. **Study the reference implementation BEFORE coding.** The AIS demod failed because we didn't understand ais-catcher's pipeline (channel separation, brute-force timing, CIC5 decimation) before writing code.
2. **Test with real signal early.** Not in a server room with no antenna.
3. **pos_reliable/receiver_id=0** — edge mode with single SDR has different assumptions than aggregator with 2500 feeders. Must be parameterized.
4. **Tag binaries with version+git hash.** Deployed wrong binary multiple times before adding /health version.

## Plan for Tomorrow

### Priority 1: Fix positions on edge (30 min)
- The 7/52 positions is because pos_reliable=1 still needs TWO global CPR decodes (even+odd pair = 1 increment, need 1 more). With 52 aircraft sharing 7.2 msg/s, some aircraft only get 1 pair in 60s.
- **Fix:** For edge mode with single receiver, publish on FIRST successful global decode (pos_reliable=0). No risk of cross-receiver contamination.
- **Result:** Should match readsb's ~40 aircraft with positions.

### Priority 2: Fix AIS 162 MHz demod (2-3 hours)
